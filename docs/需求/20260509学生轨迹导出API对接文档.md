# 学生痕迹导出 API 对接文档

## 1. 接口信息

- 接口名称：学生痕迹导出
- 请求方式：`GET`
- 请求路径：`/api/v1/export/student-tracks`
- 鉴权方式：登录态 + 权限控制
- 按钮权限：`export:studentScore:download`
- 适用角色：后台管理员；`superadmin` 可直接使用

## 2. 功能说明

该接口用于导出学生人机交互痕迹数据，支持两种模式：

1. 指定 `examNo`：导出单个学生的 Excel 文件（`.xlsx`）
2. 不指定 `examNo`：按学校拆分导出 ZIP 压缩包（`.zip`），每个学校一个 Excel 文件

`schoolCode` 可作为附加过滤条件：

- 单学生模式下：限制该学生必须属于指定学校
- 批量模式下：仅导出指定学校的数据，但返回格式仍然是 `.zip`

## 3. 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `batchCode` | `string` | 是 | 批次编码。仅允许字母、数字、下划线、短横线 |
| `examNo` | `string` | 否 | 考生号；有值时导出单个学生 |
| `schoolCode` | `string` | 否 | 学校编码；用于学校过滤 |

## 4. 返回规则

### 4.1 单学生导出

请求示例：

```http
GET /api/v1/export/student-tracks?batchCode=szw01&examNo=TEST002
```

成功时：

- HTTP 状态码：`200`
- `Content-Type`：
  `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- 返回内容：Excel 二进制流
- 文件名示例：`学生痕迹_szw01_TEST002.xlsx`

### 4.2 批量导出

请求示例：

```http
GET /api/v1/export/student-tracks?batchCode=szw01
```

成功时：

- HTTP 状态码：`200`
- `Content-Type`：`application/zip`
- 返回内容：ZIP 二进制流
- 文件名示例：`学生痕迹_szw01.zip`

ZIP 内部内容：

- 一个学校一个 `.xlsx`
- 文件名通常使用学校名称；若学校名称为空，则回退到学校编码或“其他学校”

### 4.3 按学校过滤导出

请求示例：

```http
GET /api/v1/export/student-tracks?batchCode=szw01&schoolCode=4403050001
```

成功时：

- 返回 ZIP 文件
- ZIP 中只包含该学校的导出文件

## 5. 前端调用建议

这是一个文件下载接口，不是普通 JSON 查询接口。前端调用时需要：

- 使用 `blob` 响应类型
- 从响应头 `content-disposition` 中解析文件名
- 根据 `content-type` 判断是 Excel 还是 ZIP
- 导出可能较慢，前端请求超时建议关闭或调大

Axios 示例：

```javascript
import request from '@/utils/request'

export function downloadStudentTracks(params) {
  return request({
    url: '/api/v1/export/student-tracks',
    method: 'get',
    params,
    responseType: 'blob',
    timeout: 0
  })
}
```

页面调用示例：

```javascript
async function handleDownload(params) {
  const response = await downloadStudentTracks(params)
  const contentType = response.headers['content-type'] || ''

  if (contentType.includes('application/json')) {
    const text = await response.data.text()
    const json = JSON.parse(text)
    throw new Error(json.msg || '导出失败')
  }

  const blob = new Blob([response.data], { type: contentType })

  const disposition = response.headers['content-disposition'] || ''
  const match = disposition.match(/filename\\*=UTF-8''([^;]+)|filename=([^;]+)/)
  const rawName = decodeURIComponent((match && (match[1] || match[2] || '')).replace(/\"/g, ''))
  const fileName = rawName || '学生痕迹导出'

  const url = window.URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = fileName
  document.body.appendChild(link)
  link.click()
  document.body.removeChild(link)
  window.URL.revokeObjectURL(url)
}
```

## 6. 错误返回

如果参数非法、没有数据、或者后端导出失败，接口会返回后端统一 JSON 错误结构，而不是文件流。

注意：

- 业务异常通常会被后端统一包装成 `AjaxResult`
- 这类业务异常多数情况下仍然是 HTTP `200`
- 因此前端不能只看 HTTP 状态码，必须同时判断 `content-type` 和 JSON 中的 `code` / `msg`
- 未登录、无权限这类鉴权失败，仍可能返回 HTTP `401` / `403`

常见场景：

| 场景 | 示例返回 |
| --- | --- |
| `batchCode` 为空 | `{\"code\":500,\"msg\":\"batchCode不能为空\"}` |
| `batchCode` 格式非法 | `{\"code\":500,\"msg\":\"batchCode格式非法\"}` |
| 查询不到可导出数据 | `{\"code\":500,\"msg\":\"没有可导出的数据\"}` |

因此前端下载时建议先判断响应头：

- 如果 `content-type` 包含 `application/json`，应按错误信息处理
- 如果是 `application/zip` 或 Excel 类型，再走文件下载逻辑

## 7. 前端交互建议

### 7.1 单学生导出按钮

使用条件：

- 已知 `batchCode`
- 已知学生 `examNo`

传参：

```json
{
  "batchCode": "szw01",
  "examNo": "TEST002"
}
```

### 7.2 按学校导出按钮

使用条件：

- 已知 `batchCode`
- 已知 `schoolCode`

传参：

```json
{
  "batchCode": "szw01",
  "schoolCode": "4403050001"
}
```

### 7.3 全量导出按钮

使用条件：

- 仅需传 `batchCode`

传参：

```json
{
  "batchCode": "szw01"
}
```

## 8. 导出文件字段说明

Excel 表头如下：

1. 学籍号
2. 考生号
3. 姓名
4. 学校名称
5. 页面
6. 页面进入时间
7. 页面离开时间
8. 页面数据
9. 操作步骤
10. UI控件
11. 操作类型
12. 操作详情
13. 记录时间
14. 流程名称
15. 子模块名称
16. 步骤索引
17. 页面截图
18. 备注（截断内容）

说明：

- 同一个页面下会展开为多行操作记录
- `flow_context` 系统内部操作不会导出
- 超长文本会截断到 Excel 单元格允许长度，剩余内容写入“备注（截断内容）”
- “页面截图”当前导出为截图 URL 文本，不做图片嵌入

## 9. 对接注意事项

1. 前端不要把这个接口当普通列表接口处理，必须走文件下载逻辑
2. `examNo` 有值时返回 Excel；没有值时返回 ZIP
3. `schoolCode` 不会改变单/批量模式，只负责过滤数据范围
4. 建议按钮上加导出中状态，避免用户重复点击
5. 若要做权限控制，前端按钮权限使用 `export:studentScore:download`
