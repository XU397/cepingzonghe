# 测评登录页面管理 - 监测前端 PRD

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 生成日期 | 2026-05-10 |
| 面向团队 | 监测前端 / 学生登录端 / 测评前端 |
| 关联端 | 管理后台前端、管理后台后端 |
| 功能名称 | 登录页动态配置渲染 |

## 1. 背景与目标

管理后台新增“测评登录页面管理”后，监测前端登录页需要从后端读取当前激活配置，并据此动态展示 Logo、标题、登录框标题和密码输入框。监测前端负责最终视觉呈现，所有位置、字号、颜色、间距和响应式规则由本端 CSS 控制。

本期目标：

- 登录页初始化时读取后端当前激活配置。
- 按配置展示 Logo 图片、Logo 文字或二者组合。
- 按配置展示核心主标题文字、主标题普通文字、副标题文字。
- 按配置展示登录框标题文字。
- 按配置判断是否隐藏密码输入框。
- 密码输入框隐藏时，登录提交自动使用默认密码 `1234`。
- 配置接口异常或无激活配置时，不阻塞登录，回退到当前默认登录页内容。

## 2. 范围边界

### 2.1 本期范围

- 登录页配置 API 接入。
- 登录页渲染逻辑和默认兜底配置。
- 密码隐藏模式下的表单校验和登录提交调整。
- Logo 位置 CSS class、标题 CSS class、响应式适配。
- 接口失败、图片加载失败、配置字段缺失的容错。

### 2.2 不在本期范围

- 不开发管理配置页面。
- 不支持监测前端用户在登录页手动切换主题。
- 不在前端支持自定义 CSS 注入。
- 不改造后端账号体系；隐藏密码时仍调用现有登录接口。

## 3. 配置读取 API

接口：`GET /stu/api/login-page-config/active`

调用时机：

- 登录页组件创建或应用启动进入登录页时调用。
- 未登录状态可调用。
- 若接口支持 `ETag` 或 `cacheVersion`，可在本地缓存最后一次成功配置。

成功响应示例：

```json
{
  "code": 200,
  "msg": "ok",
  "obj": {
    "id": 1001,
    "version": 3,
    "cacheVersion": "1001-3-20260510102500",
    "logo": {
      "displayType": "image_text",
      "position": "top_center",
      "imageUrl": "/profile/login-logo/file_202605100001.png",
      "text": "综合测评",
      "imageAlt": "综合测评 Logo"
    },
    "title": {
      "highlightText": "核心素养",
      "mainText": "监测平台",
      "subtitleText": "2026年春季学生综合测评"
    },
    "loginBoxTitle": "学生登录",
    "password": {
      "hidden": true,
      "defaultPasswordPolicy": "fixed_1234"
    },
    "updatedAt": "2026-05-10 10:20:00",
    "activatedAt": "2026-05-10 10:25:00"
  }
}
```

无激活配置：

```json
{
  "code": 200,
  "msg": "ok",
  "obj": null
}
```

前端处理：`obj=null` 或请求失败时使用内置默认配置。

## 4. 前端配置模型

建议在监测前端内部归一化为：

```ts
interface LoginPageConfig {
  id?: number | string
  version?: number
  cacheVersion?: string
  logo: {
    displayType: 'none' | 'image' | 'text' | 'image_text'
    position: 'top_left' | 'top_center' | 'top_right'
    imageUrl?: string
    text?: string
    imageAlt?: string
  }
  title: {
    highlightText: string
    mainText: string
    subtitleText?: string
  }
  loginBoxTitle: string
  password: {
    hidden: boolean
    defaultPasswordPolicy: 'fixed_1234'
  }
}
```

默认配置由监测前端维护，字段必须完整，确保后端不可用时页面仍能登录。

## 5. 渲染需求

### 5.1 Logo 区域

Logo 展示规则：

| displayType | 展示 |
| --- | --- |
| none | 不展示 Logo 区域，或保留 CSS 占位由前端决定 |
| image | 仅展示图片 |
| text | 仅展示文字 Logo |
| image_text | 同时展示图片和文字 |

Logo 位置规则：

| position | CSS class 建议 | 说明 |
| --- | --- | --- |
| top_left | `login-logo--top-left` | 左上位置 |
| top_center | `login-logo--top-center` | 顶部居中位置 |
| top_right | `login-logo--top-right` | 右上位置 |

要求：

- 图片使用 `img` 标签，`alt` 优先取 `logo.imageAlt`，为空时取 `logo.text`，再为空取默认文案。
- 图片加载失败时隐藏图片或替换为默认 Logo，不影响登录表单。
- 文字 Logo 使用纯文本渲染，不使用 HTML 插入。
- Logo 字体大小、颜色、图片尺寸、间距、定位和响应式行为全部由监测前端 CSS 控制。

### 5.2 标题区域

字段映射：

| 配置字段 | 页面含义 | CSS class 建议 |
| --- | --- | --- |
| title.highlightText | 核心主标题文字 | `login-title__highlight` |
| title.mainText | 主标题普通文字 | `login-title__main` |
| title.subtitleText | 副标题文字 | `login-title__subtitle` |

渲染要求：

- 核心主标题文字和主标题普通文字位于同一标题组，可按设计拼接或分行展示。
- 核心主标题文字必须独立 class，便于实现特殊颜色或强调样式。
- 副标题为空时不渲染副标题节点或隐藏该节点。
- 所有标题以纯文本输出。

### 5.3 登录框标题

- 登录框顶部标题展示 `loginBoxTitle`。
- 字段为空或异常时使用默认文案，例如 `登录` 或当前项目默认标题。
- 标题样式由登录框 CSS 控制。

### 5.4 密码输入框隐藏

规则：

- `password.hidden=false`：展示密码输入框，登录表单校验账号和密码。
- `password.hidden=true`：隐藏密码输入框，登录表单只要求用户输入账号；提交登录时自动补充 `password='1234'`。

交互要求：

- 隐藏密码时，不展示“请输入密码”的校验错误。
- 隐藏密码时，如果页面有“记住密码”功能，需隐藏或禁用密码相关记忆逻辑，避免写入默认密码造成误解。
- 隐藏密码时，可在登录框内展示弱提示：`本次测评无需输入密码`，具体是否展示由产品和现有页面空间决定。
- 若配置接口加载中，建议先使用默认配置渲染；配置返回后再切换，避免白屏。

提交示例：

```js
const payload = {
  username: form.username,
  password: loginPageConfig.password.hidden ? '1234' : form.password
}
```

## 6. 加载与容错策略

### 6.1 初始化流程

1. 进入登录页。
2. 渲染内置默认配置，保证页面可见。
3. 异步请求 `/stu/api/login-page-config/active`。
4. 请求成功且 `obj` 有效：归一化配置并更新页面。
5. 请求成功但 `obj=null`：继续使用默认配置。
6. 请求失败：读取本地缓存的最后成功配置；缓存也不存在时继续使用默认配置。

### 6.2 本地缓存

建议缓存字段：

- `cacheVersion`
- `config payload`
- `cachedAt`

建议策略：

- 成功获取有效配置后写入 localStorage 或项目已有缓存层。
- 缓存仅作为接口失败时兜底，不作为长期权威数据。
- 缓存内容不得包含任何可变密码或敏感字段。

### 6.3 字段异常容错

- 未识别的 `displayType`：按 `none` 处理。
- 未识别的 `position`：按 `top_center` 处理。
- 标题必填字段为空：使用默认配置对应字段。
- 图片 URL 为空但展示类型要求图片：降级为文字 Logo 或隐藏 Logo。
- `password.defaultPasswordPolicy` 不是 `fixed_1234`：仍按默认配置策略处理，当前版本只支持 `fixed_1234`。

## 7. 安全要求

- 所有后端返回文本都按纯文本渲染，不使用 `innerHTML` 或等价能力。
- 不允许根据后端配置动态注入 CSS 或执行脚本。
- Logo 图片只作为静态资源加载，不拼接脚本 URL。
- 默认密码 `1234` 只在隐藏密码模式下用于现有登录接口提交，不在页面明文展示为“密码”。
- 公开配置接口失败不应泄露接口错误堆栈给用户。

## 8. CSS 与响应式要求

监测前端需维护以下基础 class 或等价结构：

```html
<div class="login-page">
  <div class="login-logo login-logo--top-center">
    <img class="login-logo__image" />
    <span class="login-logo__text"></span>
  </div>
  <div class="login-title">
    <span class="login-title__highlight"></span>
    <span class="login-title__main"></span>
    <p class="login-title__subtitle"></p>
  </div>
  <div class="login-panel">
    <h2 class="login-panel__title"></h2>
  </div>
</div>
```

样式控制原则：

- 桌面端和移动端均需保证 Logo 与标题不遮挡登录框。
- `top_left`、`top_center`、`top_right` 三类位置在小屏下可统一折叠到顶部居中，只要不影响登录即可。
- 标题过长时允许换行，不应溢出视口。
- 图片 Logo 应设置最大宽高，避免超大图片撑破布局。

## 9. 与现有登录逻辑的兼容

- 登录接口仍沿用现有账号登录接口。
- 账号字段、验证码字段、批次跳转逻辑保持不变。
- 隐藏密码模式只影响登录页表单展示和提交 payload 中 password 的来源。
- 如果当前登录页有验证码，验证码逻辑不受本配置影响。
- 如果当前登录页有“记住密码”，隐藏密码模式需关闭密码记忆；显示密码模式保持原逻辑。

## 10. 埋点与日志建议

建议记录前端调试日志或埋点事件：

| 事件 | 触发时机 | 内容 |
| --- | --- | --- |
| login_config_fetch_success | 配置接口成功 | configId、version、cacheVersion |
| login_config_fetch_failed | 配置接口失败 | errorCode、fallbackType |
| login_config_fallback_default | 使用内置默认配置 | reason |
| login_password_hidden_submit | 隐藏密码模式提交 | configId，不记录密码 |

生产环境日志不得输出默认密码字段。

## 11. 验收标准

- 后端存在激活配置时，登录页能展示配置中的 Logo、标题和登录框标题。
- Logo 展示类型为图片、文字、图片+文字、不展示时，页面行为均符合配置。
- Logo 位置为左上、居中、右上时，页面应用对应 CSS class。
- 核心主标题文字使用独立 class，可被特殊颜色样式控制。
- 副标题为空时页面不出现多余空白或异常符号。
- `password.hidden=false` 时，密码输入框正常显示并参与校验。
- `password.hidden=true` 时，密码输入框隐藏，账号可使用默认密码 `1234` 提交登录。
- 配置接口失败或返回空配置时，页面仍使用默认登录页并可正常登录。
- 图片加载失败不影响账号输入和登录提交。
- 所有配置文本均未以 HTML 方式执行。

## 12. 测试用例建议

### 12.1 功能用例

1. 返回 `image_text + top_center + hidden=true`，校验图片、文字、标题和隐藏密码提交。
2. 返回 `image + top_left + hidden=false`，校验密码输入框展示和手动密码提交。
3. 返回 `text + top_right`，校验文字 Logo 和位置 class。
4. 返回 `none`，校验 Logo 区域不影响布局。
5. 返回 `obj=null`，校验默认配置兜底。
6. 接口 500 或网络失败，校验缓存或默认配置兜底。
7. Logo 图片 URL 404，校验图片降级。
8. 标题包含 `<script>` 字符串，校验按文本展示且不执行。

### 12.2 回归用例

- 原有账号密码登录成功流程不变。
- 验证码刷新和校验逻辑不变。
- 登录成功后的测评跳转逻辑不变。
- 移动端登录页不出现横向滚动条。

## 13. 推荐实施顺序

1. 增加登录页配置 API 客户端和默认配置。
2. 在登录页初始化流程中接入异步配置读取。
3. 改造 Logo、标题、登录框标题渲染结构。
4. 改造密码隐藏模式下的表单校验与提交 payload。
5. 补齐 CSS class、图片降级和响应式适配。
6. 增加缓存兜底、异常日志和测试用例。
7. 与后端和管理后台前端完成三端联调。
