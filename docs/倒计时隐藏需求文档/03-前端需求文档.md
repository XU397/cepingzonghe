## 一、需求背景

### 1.1 业务需求

在某些线下考试场景中，需要隐藏学生端的倒计时显示：
- 由监考人员人工控制考试时间
- 减少学生看到倒计时的焦虑感
- 倒计时逻辑本身不变，只是UI不显示

### 1.2 技术方案

前端需要：
1. 从登录响应中读取 `displayOptions.hideTimerDisplay` 配置
2. 将配置存储到 Context 或 localStorage
3. Timer 组件根据配置决定是否渲染

## 三、详细设计

### 3.1 AppContext 扩展

**文件**：`src/context/AppContext.jsx`

```jsx
// 新增状态
const [displayOptions, setDisplayOptions] = useState(() => {
  // 从 localStorage 恢复
  const stored = localStorage.getItem('hci-displayOptions');
  if (stored) {
    try {
      return JSON.parse(stored);
    } catch {
      return { hideTimerDisplay: false };
    }
  }
  return { hideTimerDisplay: false };
});

// 新增更新方法
const updateDisplayOptions = useCallback((options) => {
  setDisplayOptions(prev => {
    const next = { ...prev, ...options };
    localStorage.setItem('hci-displayOptions', JSON.stringify(next));
    return next;
  });
}, []);

// 新增便捷 getter
const shouldHideTimer = useMemo(() => {
  return displayOptions?.hideTimerDisplay === true;
}, [displayOptions]);

// Provider value 新增
const contextValue = {
  // ... 现有字段
  displayOptions,
  updateDisplayOptions,
  shouldHideTimer,
};
```

### 3.2 LoginPage 修改

**文件**：`src/pages/LoginPage.jsx`

```jsx
// 在登录成功回调中
const handleLoginSuccess = (response) => {
  const { obj } = response;

  // ... 现有逻辑

  // 新增：解析并存储 displayOptions
  if (obj.displayOptions) {
    updateDisplayOptions(obj.displayOptions);
```

### 3.3 Timer 组件封装

建议创建一个统一的 Timer 显示组件，封装隐藏逻辑：

**文件**：`src/shared/ui/TimerDisplay/TimerDisplay.jsx`

```jsx
import { useAppContext } from '@/context/AppContext';

/**
 * 统一的倒计时显示组件
 * 根据 displayOptions.hideTimerDisplay 配置决定是否显示
 *
 * @param {number} remaining - 剩余时间（秒）
 * @param {string} type - 计时器类型 'task' | 'questionnaire' | 'notice'
 * @param {string} className - 自定义样式类
 * @param {boolean} forceShow - 强制显示（忽略隐藏配置）
 */
export const TimerDisplay = ({
  remaining,
  type = 'task',
  className,
  forceShow = false,
}) => {
  const { shouldHideTimer } = useAppContext();

  // 配置隐藏且非强制显示时，不渲染
  if (shouldHideTimer && !forceShow) {
    return null;
  }

  // 格式化时间
  const minutes = Math.floor(remaining / 60);
  const seconds = remaining % 60;
  const timeString = `${minutes.toString().padStart(2, '0')}:${seconds.toString().padStart(2, '0')}`;

  // 样式类
  const timerClasses = [
    'timer-display',
    `timer-${type}`,
    remaining < 60 && 'timer-critical',
    remaining < 300 && remaining >= 60 && 'timer-warning',
    className,
  ].filter(Boolean).join(' ');

  return (
    <div className={timerClasses}>
      <span className="timer-icon">⏱️</span>
      <span className="timer-value">{timeString}</span>
    </div>
  );
};

export default TimerDisplay;
```

### 3.4 现有 Timer 组件迁移

对于已有的 Timer 组件，有两种迁移策略：

#### 策略A：使用统一组件替换（推荐）

```jsx
// 修改前
<div className="timer">{formatTime(remaining)}</div>

// 修改后
<TimerDisplay remaining={remaining} type="task" />
```

#### 策略B：在现有组件中添加条件判断

```jsx
// 在现有 Timer 组件中
const { shouldHideTimer } = useAppContext();

if (shouldHideTimer) {
  return null;
}

// ... 原有渲染逻辑
```

## 七、注意事项

### 7.1 不要修改倒计时逻辑

**重要**：本次修改只涉及 UI 显示，不修改任何倒计时逻辑：
- TimerService 正常运行
- 超时回调正常触发
- 时间记录正常保存

### 7.2 向后兼容

- 后端未返回 `displayOptions` 时，使用默认值
- 不影响现有功能

### 7.3 清理策略

登出时需要清理 `hci-displayOptions`：

```jsx
const handleLogout = () => {
  // ... 现有清理逻辑
  localStorage.removeItem('hci-displayOptions');
};
```
