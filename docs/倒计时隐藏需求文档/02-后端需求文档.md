# 后端需求文档：倒计时显示控制功能

> 版本：1.0.0
> 日期：2025-12-11
> 状态：待实施

---

## 一、需求背景

### 1.1 业务需求

在某些线下考试场景中，需要隐藏学生端的倒计时显示，由监考人员人工控制考试时间。

### 1.2 技术方案

后端需要：
1. 新增配置字段存储"是否隐藏倒计时"
2. 在学生登录接口中返回该配置
3. 提供管理端配置接口

---

## 二、数据库设计

### 2.1 新增字段

建议在以下表中新增字段（根据实际表结构选择）：

#### 方案A：Flow配置表

```sql
ALTER TABLE flow_config ADD COLUMN hide_timer_display TINYINT(1) DEFAULT 0 COMMENT '是否隐藏学生端倒计时显示 0-显示 1-隐藏';
```

#### 方案B：批次配置表

```sql
ALTER TABLE batch_config ADD COLUMN hide_timer_display TINYINT(1) DEFAULT 0 COMMENT '是否隐藏学生端倒计时显示 0-显示 1-隐藏';
```

#### 方案C：通用配置表（JSON字段）

```sql
-- 如果已有 display_options JSON 字段
-- 在 JSON 中新增 hideTimerDisplay 属性
UPDATE flow_config SET display_options = JSON_SET(COALESCE(display_options, '{}'), '$.hideTimerDisplay', false);
```

### 2.2 字段说明

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `hide_timer_display` | TINYINT(1) / Boolean | 0 / false | 0=显示倒计时，1=隐藏倒计时 |

## 三、接口设计

### 3.1 学生登录接口（修改）

**接口**：`POST /stu/login`

**修改点**：在响应中新增 `displayOptions` 字段

**响应示例**：
```json
{
  "code": 200,
  "msg": "登录成功",
  "obj": {
    "batchCode": "250619",
    "examNo": "1001",
    "url": "/flow/assessment-001",
    "pageNum": "1",
    "studentName": "张三",
    "displayOptions": {
      "hideTimerDisplay": true
    }
  }
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `displayOptions` | Object | 否 | 显示选项配置，不存在时前端使用默认值 |
| `displayOptions.hideTimerDisplay` | Boolean | 否 | 是否隐藏倒计时，默认 `false` |

**配置查找逻辑**：
```
1. 根据学生将要进入的 flowId 查询 Flow 级别配置
2. 如果 Flow 级别未配置，查询 batchCode 对应的批次配置
3. 如果都未配置，返回 null 或不返回该字段（前端使用默认值 false）
```

### 3.2 管理端配置接口（新增）

#### 3.2.1 保存配置

**接口**：`POST /admin/display-options/save`

**请求体**：
```json
{
  "configType": "flow",           // "flow" 或 "batch"
  "configId": "assessment-001",   // flowId 或 batchCode
  "displayOptions": {
    "hideTimerDisplay": true
  }
}
```

**响应**：
```json
{
  "code": 200,
  "msg": "保存成功"
}
```

#### 3.2.2 查询配置

**接口**：`GET /admin/display-options/get`

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `configType` | String | 是 | "flow" 或 "batch" |
| `configId` | String | 是 | flowId 或 batchCode |

**响应**：
```json
{
  "code": 200,
  "obj": {
    "configType": "flow",
    "configId": "assessment-001",
    "displayOptions": {
      "hideTimerDisplay": true
    }
  }
}
```

## 四、接口兼容性

### 4.1 向后兼容

- `displayOptions` 字段为**可选字段**
- 旧版前端不识别该字段，不影响正常使用
- 新版前端在字段缺失时使用默认值 `false`（显示倒计时）

### 4.2 版本迭代

| 版本 | 行为 |
|------|------|
| 旧后端 + 旧前端 | 正常，显示倒计时 |
| 旧后端 + 新前端 | 正常，显示倒计时（前端使用默认值） |
| 新后端 + 旧前端 | 正常，旧前端忽略新字段 |
| 新后端 + 新前端 | 正常，根据配置显示/隐藏倒计时 |

## 五、测试用例

### 5.1 登录接口测试

| 用例 | 输入 | 预期输出 |
|------|------|----------|
| 未配置 | 正常登录 | `displayOptions` 为空或不返回 |
| Flow配置隐藏 | 登录进入已配置Flow | `displayOptions.hideTimerDisplay = true` |
| 批次配置隐藏 | 登录进入已配置批次 | `displayOptions.hideTimerDisplay = true` |
| Flow和批次都配置 | 登录 | 返回Flow级别配置（优先级更高） |

### 5.2 管理接口测试

| 用例 | 操作 | 预期结果 |
|------|------|----------|
| 保存配置 | POST 保存 hideTimerDisplay=true | 返回成功，数据库更新 |
| 查询配置 | GET 查询 | 返回已保存的配置 |
| 查询不存在配置 | GET 查询未配置的ID | 返回空或默认值 |

2. **安全**：管理接口需验证管理员权限
