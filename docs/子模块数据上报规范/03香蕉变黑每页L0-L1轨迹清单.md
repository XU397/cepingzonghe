# 香蕉变黑每页 L0 / L1 轨迹记录清单

本文是 `01轨迹上报规范.md` 的内容层补充，目标是回答一个更前置的问题：香蕉变黑子模块每一页到底有哪些内容、控件和学生行为需要被记录，避免只定义事件格式而漏采关键认知过程证据。

依据材料：

- `01轨迹上报规范.md`
- `香蕉变黑两套标准文档（终稿）.pdf`
- `Page01.PNG` 至 `Page13.PNG`

## 1. 分层口径

为避免和旧文档中的 L0 / L1 叫法混淆，本清单采用当前工程链路口径：

| 层级 | 本清单中的含义 | 是否进入上报 | 说明 |
|---|---|---:|---|
| L0 原始采集项 | 前端内部可观察到的浏览器事件、控件操作、页面状态变化 | 否，需先去噪/聚合 | 例如滚动、悬停、聚焦、输入、失焦、点击、参数改变、页面隐藏/可见 |
| L1 内容与控件语义层 | 把 L0 绑定到稳定的页面、内容、字段、题目、选项、方案行语义 | 否，作为 L2 生成依据 | 例如 `field_id=input_question_1`、`content_id=chat_bubble_02_01`、`question_index=2`、`row_id=...` |
| L2 标准事件层 | 前端写入 `operationList` 的标准事件 | 是 | 例如 `TEXT_BLUR`、`OPEN_MODAL`、`EXECUTE_EXP`、`SUBMIT_ATTEMPT` |
| L3 行为标签层 | 后端根据 L2 时间序列生成的过程标签 | 否，前端不得生成 | 例如 `GENERATE_THINK`、`CHECK_REVIEW`、`CHART_INSPECT` |

旧 PDF 中“轨迹数据 -> 打标转换标准（L1）”描述的很多行为标签，在当前 `01/02` 规范中应理解为后端 L3 候选标签或页面级策略辅助信息。本文每页仍列出“可支撑的后端标签”，但它们不是前端 L1，也不能由前端直接上报。

## 2. 全局必守规则

- 所有页面都必须有稳定 `page_id`、`page_index`、`page_type`、毫秒级时间、页面内递增 `event_seq/code`、唯一 `trace_id`。
- `START_PAGE` 记录页面到达、初始状态、registry 版本；`PAGE_HIDDEN` / `PAGE_VISIBLE` 只在切 tab、窗口失焦等条件下上报，用于避免把离开页面的时间误算为思考。
- 所有 `SUBMIT_ATTEMPT` 必须记录 `submit_attempt_id`、`validation_status=success/blocked`、`missing_fields`、`error_msg`、`submit_trigger`；不得把校验状态塞进 `value_after`。
- 文本输入不得逐字上报；`TEXT_CHANGE` 只保留防抖/节流后的过程特征，最终完整文本只在 `TEXT_BLUR.value_after` 落盘。
- 系统自动动画、自动出现的气泡、自动渲染的实验结果、无意义空白点击、极短划过、重复双击都不是独立学生认知证据。
- L1 语义绑定必须使用 Field Registry / Content Registry / Question Registry，不允许在事件里动态拼中文语义，也不允许把香蕉主题词写入通用字段名。
- 生命周期事件和页面完成状态进入 `pageMetadata.lifecycle`，不进入 HMM 主观测序列；策略标记进入 metadata/strategyFlags，默认也不进入 HMM。

### 2.1 页面主说明文字与长停顿采集原则

本次讨论后补充一个关键边界：香蕉变黑多数页面的说明文字、题干、任务要求在页面加载后就是主内容，默认完整或基本完整可见。因此，默认可见的主说明文字不宜每次单独生成 `CONTENT_EXPOSE`，否则只是在重复记录“系统把文字显示出来了”，行为信息量较低。

推荐口径：

| 对象 | 记录方式 | 是否作为学生行为证据 | 说明 |
|---|---|---:|---|
| 页面初始可见的主说明文字、题干、约束提示 | 写入 `START_PAGE.metadata.initial_visible_content_ids` | 否 | 表示学生进入页面时有阅读机会，不表示已经阅读或理解 |
| 页面加载后到首次有效操作前的长时间无操作 | 聚合为 `PAGE_IDLE` | 是，弱到中等置信度 | 可作为阅读、理解任务、思考或犹豫的候选证据 |
| 提交被拦截后的长时间无操作 | 聚合为 `PAGE_IDLE`，`idle_phase=after_blocked_submit` | 是，弱到中等置信度 | 可作为重新理解规则、检查缺失项或补救前停顿的候选证据 |
| 需要滚动/展开/弹窗/tooltip 才能看到的内容 | `CONTENT_EXPOSE` / `CONTENT_ACTIVATE` / `CONTENT_VIEW` | 是，注意代理证据 | 例如聊天旧气泡、资料卡、折叠说明、图表 tooltip、被小屏遮挡的说明 |

`PAGE_IDLE` 推荐只在页面可见、窗口聚焦、无有效交互持续达到阈值时生成。推荐 metadata：`idle_duration_ms`、`idle_phase`、`idle_start_event`、`idle_end_event`、`page_visible`、`window_focused`、`initial_visible_content_ids`、`next_event_type`、`cursor_region`。它不能直接命名为“阅读”或“理解”，只能为后端推断 `ORIENT_PAGE`、`READ_OR_THINK_CANDIDATE`、`PRE_ACTION_PLANNING`、`POST_BLOCKED_REORIENT` 等候选状态提供证据。

## 5. 面向 HMM 与认知分析的保真/去噪要求

### 5.1 必须保留的时序

- 所有核心作答页：`START_PAGE -> PAGE_IDLE -> 首次有效操作` 的加载后定向/阅读/思考候选序列。
- Page02：聊天查看、输入开始、输入变化、修正、提交/拦截、拦截后回看。
- Page03：资料卡打开/关闭、资料停留、多选勾选/取消、作答后回看。
- Page05：三个方案字段的聚焦顺序、输入/修改顺序、跨字段切换。
- Page06：方法槽位与优/缺点维度的切换、评价覆盖情况。
- Page08：是否设置参数、是否执行实验、是否重置、是否直接跳过。
- Page09-11：实验操作与答案选择的先后关系、作答后是否回实验区核对。
- Page12：图表查看、方案行构造、最佳方案选择、理由输入之间的先后关系。

### 5.2 必须过滤或降权的噪音

| 噪音类型 | 处理规则 |
|---|---|
| 空白点击、鼠标移动 | 不进入 L2 |
| 默认可见主说明文字 | 不单独生成 `CONTENT_EXPOSE`；只进入 `START_PAGE.metadata.initial_visible_content_ids` |
| 页面不可见或窗口失焦期间的无操作 | 不生成有效 `PAGE_IDLE`，或标记为离开/分心候选，不作为阅读思考证据 |
| 极短 hover / 误触打开 | 低于阈值丢弃或仅作为 raw 计数，不生成阅读/查看证据 |
| 连续滚动逐像素事件 | 聚合为一个 `CHAT_SCROLL` 或内容查看段 |
| 文本逐字输入 | 防抖/节流为 `TEXT_CHANGE`，最终文本由 `TEXT_BLUR` 上报 |
| 输入法 composition 中间态 | 不独立上报，只在稳定文本变化后采集 |
| 执行实验按钮双击 | 防双击去重，窗口内重复点击标记 `click_debounce_applied=true` |
| 系统自动渲染结果 | 只作为 `EXECUTE_EXP.metadata.result_snapshot`，不是学生行为 |
| 页面生命周期事件 | 只进生命周期 metadata，不进 HMM 主观测序列 |

### 5.3 前端绝不能直接上报的内容

- 能力判断：如“学生具备信息整合能力”“学生方案质量高”。
- 策略结论：如“深度阅读策略”“高质量证据推理策略”。
- 后端 L3 行为标签：如 `GENERATE_THINK`、`CHECK_REVIEW`、`ARGUMENTATION`。
- L4 聚合特征：如字段完成度、关键证据覆盖率、总阅读时长，除非后续版本明确下放到前端。

## 6. 验收清单

- 所有页面的默认主说明/题干/约束应进入 `START_PAGE.metadata.initial_visible_content_ids`，不单独生成默认 `CONTENT_EXPOSE`。
- 达到阈值的页面加载后长停顿应生成 `PAGE_IDLE`，并记录 `idle_phase`、`idle_duration_ms`、`page_visible`、`window_focused`、`idle_end_event/next_event_type`。
- Page02 必须有 `input_question_1`，聊天气泡进入 Content Registry，自动气泡展示不作为主动阅读。
- Page03 必须记录资料卡打开/关闭和多选勾选/取消，不能只保留最终选项。
- Page05 必须分别记录 `input_idea_1`、`input_idea_2`、`input_idea_3`。
- Page06 必须分别记录 6 个 `method_<slot>_<side>` 字段，并保留 `method_slot` 与 `evaluation_side`。
- Page08 禁止出现 `SELECT_ANSWER`。
- Page09-11 每页只记录当前页当前题，必须有 `question_group_id`、`question_index`、`question_id`、`option_id`。
- Page12 必须有 `chart_evidence_1`、稳定 `row_id`、`plan_param_1/2`、`SELECT_BEST`、`reason_text`。
- Page13 不进入 HMM 核心探究序列，只触发任务结束和全局汇总。
- 所有 blocked 提交必须带 `missing_fields`，被拦截后的补救行为必须能从后续事件时序中恢复。
